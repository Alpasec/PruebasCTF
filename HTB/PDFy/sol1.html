Attempting File Leaks via various HTML tags:

1. Using <object> tag:

This tries to embed the file as a generic object.

2. Using <embed> tag:

Similar to object, but sometimes handled differently.

3. Using <img> tag (less likely for text, but good for testing):

If the file content is interpreted as an image (e.g., text might show as broken image, but still processed).

4. Using <link> tag for stylesheet (can sometimes expose content):

If the file is loaded as a stylesheet, its content might appear in parsing errors or console, sometimes rendered.

5. Using <script> tag (less likely for text file, but possible for JS or error messages):

If the file is loaded as a script, its content might generate errors in the PDF's internal console or reveal parts of the file.

Still includes this, just in case the server is on EC2 and this specific tag works.
